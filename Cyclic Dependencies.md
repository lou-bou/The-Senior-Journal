Experience - I once tried to make a todo list project in vanilla JS for the first time. I hadn't planned the project structure and architecture. I dived in immediately and started coding the classes. After getting the project 90% done, I had a couple of missing features and some bugs, when trying to fix them, it felt impossible, because the project was filled with cyclic dependencies which I made mindlessly importing modules without planning.

Lesson - Always plan a project's structure and architecture beforehand. Pretty obvious, but it's an important lesson.
Also, always take a look at your dependency graph while building your project (some IDEs offer this as an extension like vscode).
